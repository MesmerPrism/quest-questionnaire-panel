Quest Questionnaire Panel

A native Quest panel for Big Red Button study prompts.

The panel app owns foreground 2D questionnaire screens. The Big Red Button Unity app keeps ownership of the 3D button, condition runtime, heartbeat-driven visuals, and physical press counting.

Protocol quest.questionnaire.v1
Result channel caller-owned content URI
Completion signal broadcast PendingIntent

Functional Demos

These clips were captured from a Quest headset with CLI-driven triggers. The debug commands call the same app paths used by visible controls, so they are useful for repeatable validation.

Heartbeat blink

Unity can start a timed blink on the Big Red Button cap. This is the visual hook for heartbeat-connected state once live signal data is attached. 

adb shell am start -W \
  -n org.thebigredbuttoninstitute.app/com.unity3d.player.UnityPlayerGameActivity \
  --es brb.runtimeCommand "blink_button:5"

Button press and counter

The runtime press command plays the imported press animation and increments the world-space counter in Unity. 

adb shell am start -W \
  -n org.thebigredbuttoninstitute.app/com.unity3d.player.UnityPlayerGameActivity \
  --es brb.runtimeCommand press_button

Open the panel

Unity launches the native 2D Quest panel through an explicit Android intent while keeping the 3D scene alive behind it. 

adb shell am start -W \
  -n org.thebigredbuttoninstitute.app/com.unity3d.player.UnityPlayerGameActivity \
  --es brb.questionnaireTrigger final

Navigate questionnaire screens

Debug command scripts can move through the panel at human reasonable speed for headset captures and regression checks. 

adb shell am start -W \
  -n org.thebigredbuttoninstitute.app/com.unity3d.player.UnityPlayerGameActivity \
  --es brb.questionnaireTrigger initial \
  --es brb.questionnaireCommandScript "language:en-US,next,participant_code:QA001,age:30,next,prior:no,next" \
  --ei brb.questionnaireCommandIntervalMs 1400

Submit and close

The panel writes JSON to the granted result URI, sends the caller callback, and finishes only its own activity. 

adb shell am start -W \
  -n org.thebigredbuttoninstitute.app/com.unity3d.player.UnityPlayerGameActivity \
  --es brb.questionnaireTrigger final \
  --es brb.questionnaireCommandScript "final:1,next,submit" \
  --ei brb.questionnaireCommandIntervalMs 1300

Split-App Boundary

The app boundary keeps questionnaire collection out of Unity while preserving Unity as the runtime authority for the 3D study world.

Unity BRB app owns

  • 3D Big Red Button object and press animation
  • Button press counting and final physical interaction
  • Condition sessions and heartbeat-linked runtime visuals
  • Caller-owned result URI creation and result validation

Panel app owns

  • Foreground 2D Quest questionnaire screens
  • Localized prompt presentation and answer collection
  • Writing result JSON to the granted URI
  • Sending the caller-provided completion PendingIntent

Command Structure

Runtime commands drive Unity. Questionnaire commands are forwarded by Unity to the panel and are debug-only in the panel app.

Unity runtime command

--es brb.runtimeCommand press_button
--ei brb.runtimeCommandRepeat 3
--ei brb.runtimeCommandIntervalMs 700

Unity runtime script

--es brb.runtimeCommandScript "center_button,blink_button:6,press_button,status"
--ei brb.runtimeCommandIntervalMs 700

Panel route

--es brb.questionnaireTrigger initial
--es brb.questionnaireCommandScript "language:en-US,next,participant_code:QA001,age:30,next,prior:no,next"
--ei brb.questionnaireCommandIntervalMs 1400
Required extra Purpose
session_id Caller session identifier.
request_id Opaque per-request identifier validated on return.
request_nonce Random nonce binding request and result.
request_json JSON matching the versioned request schema.
result_uri Caller-owned writable content:// URI.
return_to_caller Broadcast PendingIntent fired after writing JSON.

Communication Flow

The panel never writes answers to public storage and never returns answer payloads in callback extras. The callback is only a signal.

  1. Caller prepares request Persist pending state, create nonce and private result file.
  2. Caller launches panel Explicit component, request JSON, write-granted result URI.
  3. Panel collects answers Render the requested BRB stage sequence in native 2D UI.
  4. Panel writes result JSON Close the stream before signaling completion.
  5. Caller validates result Check protocol, request id, nonce, schema, status, and stage. 
{
  "protocol_version": "quest.questionnaire.v1",
  "request_id": "7c3f4d2c...",
  "nonce": "caller-generated-random-value",
  "questionnaire_id": "brb.study",
  "stage": "final:end_confirmation",
  "status": "completed",
  "answers": {
    "final_rating": 1,
    "extra_presses_prompt_acknowledged": true
  }
}
